01 Objetivo de la sesión

En esta sesión cada equipo recibirá un proyecto Python base con una API REST construida con FastAPI. El proyecto tiene deuda técnica intencional visible en SonarQube. El trabajo del equipo es triple:

① Escribir

Completar el Jenkinsfile con los 7 TODOs marcados y las pruebas unitarias faltantes hasta superar el 85% de cobertura.

② Analizar

Ejecutar el pipeline, leer el dashboard de SonarQube e identificar cada issue reportado clasificándolo por tipo y severidad.

③ Corregir

Aplicar al menos 3 correcciones técnicas sobre la deuda detectada, re-ejecutar el pipeline y verificar que los issues desaparezcan.

Tiempo estimado: 2 horas. Distribución sugerida: 20 min fork y setup · 30 min Jenkinsfile · 40 min pruebas · 30 min análisis SonarQube y correcciones.

02 Fork y descarga del proyecto

Cada equipo trabaja sobre su propio fork del repositorio base. Sigan estos pasos exactamente usando la consola — no usen GitHub Desktop ni interfaz gráfica.

Paso 1 — Hacer Fork en GitHub

1

Abrir el repositorio base en el navegador

Ingresa a: https://github.com/duvanfr/software_quality_project

2

Crear el fork

Clic en el botón Fork (esquina superior derecha) → elige tu cuenta personal → clic en Create fork. Esto crea una copia del repositorio bajo tu usuario de GitHub.

Paso 2 — Clonar el fork por consola

En Windows: usa PowerShell. En Mac/Linux: usa el Terminal. Reemplaza TU-USUARIO con tu nombre de usuario de GitHub.
Consola Comandos Git paso a paso 
# 1. Verificar que git está instalado
git --version

# 2. Clonar tu fork (reemplaza TU-USUARIO con tu usuario GitHub)
git clone https://github.com/TU-USUARIO/software_quality_project.git

# 3. Entrar a la carpeta del proyecto
cd software_quality_project

# 4. Verificar que estás en la rama main y conectado a tu fork
git remote -v
# Debes ver: origin  https://github.com/TU-USUARIO/software_quality_project.git

# 5. Agregar el repositorio original como upstream (para actualizaciones futuras)
git remote add upstream https://github.com/duvanfr/software_quality_project.git

# 6. Crear una rama de trabajo con el nombre del equipo
git checkout -b feature/equipo-01
# Reemplaza "equipo-01" con el identificador de tu equipo

# 7. Confirmar que la rama fue creada
git branch

Antes de completar el Jenkinsfile asegúrate de que Jenkins y SonarQube están corriendo con los comandos de clase anterior, y que el job de Jenkins apunta al fork de tu equipo.

Una vez el pipeline pase el Stage 4, SonarQube habrá analizado el código. Abre http://localhost:9000 y navega al proyecto.

Corrección obligatoria 1 — División por cero

El bug más grave es la división por cero en academic_service.py. La corrección es simple:

Python ❌ Vulnerable 
def calcular_promedio_estudiante(codigo):
    notas = [n for n in get_notas()
             if n["codigo_estudiante"] == codigo]
    total = sum(n["valor"] for n in notas)
    return round(total / len(notas), 2)  # ← crash
Python ✅ Corregido 
def calcular_promedio_estudiante(codigo):
    notas = [n for n in get_notas()
             if n["codigo_estudiante"] == codigo]
    if not notas:
        return 0.0
    return round(sum(n["valor"] for n in notas) / len(notas), 2)

Corrección obligatoria 2 — Magic number como constante

Python ❌ Magic number 
def es_aprobado(nota: float) -> bool:
    return nota >= 3.0  # ← repetido 4 veces
Python ✅ Constante nombrada 
NOTA_MINIMA_APROBACION: float = 3.0

def es_aprobado(nota: float) -> bool:
    return nota >= NOTA_MINIMA_APROBACION
